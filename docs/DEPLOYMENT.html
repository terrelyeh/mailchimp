1. 架構總覽

Mailchimp Multi-Region Dashboard 管理 6 個區域（US / EU / APAC / JP / TW / INDIA）的電子報行銷數據。系統由同一個 GitHub repository 部署為兩個獨立的 Vercel projects，搭配 Supabase 作為資料庫。

┌──────────┐ HTTPS ┌──────────────────────────────────┐ │ │ ──────────────▶ │ Vercel Frontend │ │ 使用者 │ │ edm-dashboard-eg.vercel.app │ │ (Browser) │ │ React 19 + Vite + Tailwind │ │ │ │ Root Dir: frontend/ │ └──────────┘ └────────────────┬─────────────────┘ │ │ API Requests /api/* │ ▼ ┌──────────────────────────────────┐ │ Vercel Backend (Serverless) │ │ edm-dashboard-api.vercel.app │ │ Python FastAPI │ │ Root Dir: backend/ │ └───────────────┬──────────┬───────┘ │ │ REST API │ │ HTTP (API Key) ▼ ▼ ┌────────────────┐ ┌────────────────┐ │ Supabase │ │ Mailchimp │ │ PostgreSQL │ │ API v3.0 │ │ ap-northeast-1 │ │ 6 Regions │ └────────────────┘ └────────────────┘
ℹ
同一 GitHub Repo，兩個 Vercel Projects
Repository terrelyeh/mailchimp 同時包含前端與後端程式碼。兩個 Vercel projects 各自指向不同的 Root Directory（frontend/ 與 backend/）。前端 push 自動部署，後端需手動執行 vercel --prod。

3. 部署流程

3.1 Frontend（自動部署）

git push origin main → Vercel 偵測變更 → npm install && build → 部署至 CDN → ~1 min 上線 
# 1. 修改前端程式碼後 commit & push
git add .
git commit -m "feat: your changes"
git push origin main

# 2. Vercel 自動觸發部署，約 1 分鐘後完成
# 3. 前往 https://edm-dashboard-eg.vercel.app 確認

3.2 Backend（手動部署）

# 1. 進入 backend 目錄
cd backend

# 2. 執行 Vercel 部署
vercel --prod

# 3. 部署完成後會顯示 Production URL
⚠
修改環境變數後必須重新部署
在 Vercel Dashboard 修改 Environment Variables 後，需要重新執行 vercel --prod 才會生效。

3.3 Database（無需部署）

Schema 變更直接在 Supabase Dashboard 的 SQL Editor 執行，不需要額外的部署步驟。

# Supabase SQL Editor
https://supabase.com/dashboard/project/mxfwbewanezlyvyxuhyt/sql

5. 常見問題 / 注意事項

Vercel Deployment Protection 需關閉

後端 Vercel project 必須關閉 Deployment Protection。否則所有 API 請求會被導向 Vercel 認證頁面，前端無法呼叫 API。

設定路徑：Vercel Dashboard → Project Settings → General → Deployment Protection → Off

Supabase Pooler "Tenant not found" 問題

Supabase connection pooler（Transaction / Session mode）目前有 "Tenant or user not found" 錯誤，無法使用 psycopg2 直接連線。已改用 supabase-py REST API 連線，不受此問題影響。

Vercel Serverless 10 秒 Timeout

Free Tier 的 Serverless Functions 有 10 秒執行時間限制。首次同步大量 Mailchimp 資料或同時抓取所有區域時可能超時。

因應策略：

  • 使用 ThreadPoolExecutor 平行抓取多區域
  • 分區域同步，避免一次全部抓取
  • 考慮升級 Vercel Pro（60 秒 timeout）

bcrypt 版本相容性

passlib 不相容 bcrypt 5.x。requirements.txt 中已限制 bcrypt<5，請勿升級。

Supabase Free Tier 自動暫停

Free Tier 在 7 天無活動後會自動暫停資料庫。已設定 keep-alive cron job 定期 ping 防止暫停。若已暫停，可在 Supabase Dashboard 手動 "Restore project" 恢復。

CORS 設定

後端 main.py 中的 EXTRA_ORIGINS 列表必須包含所有前端網域。目前已包含：

  • https://edm-dashboard-eg.vercel.app
  • https://mailchimp-dashboard.vercel.app
  • http://localhost:5173
  • http://localhost:3000

新增前端網域時需同步更新此列表並重新部署後端。

Mock Data 模式

若未設定任何 Mailchimp API key，系統自動使用 mock data。不需要真實的 Mailchimp 帳號即可啟動完整系統，方便本地開發和測試。

6. 維護指南

6.1 新增一個區域

1

設定 Mailchimp API Key

在 Vercel Backend project（edm-backend）新增環境變數：

MAILCHIMP_API_KEY_NEWREGION=your-api-key-here
MAILCHIMP_SERVER_PREFIX_NEWREGION=us1
2

重新部署後端

cd backend
vercel --prod

系統會自動偵測以 MAILCHIMP_API_KEY_ 為前綴的環境變數，將新區域加入可用區域列表。前端無需修改。

6.2 檢查 Supabase 資料庫

方法一：Supabase Dashboard

  1. 前往 Supabase Dashboard
  2. 左側選單 → Table Editor（瀏覽資料）或 SQL Editor（執行查詢）

方法二：透過 API

# 檢查 cache 統計
curl https://edm-dashboard-api.vercel.app/api/diagnostics

# 檢查健康狀態
curl https://edm-dashboard-api.vercel.app/api/health

6.3 查看 Vercel Logs

Dashboard 方式：

  1. 前往 vercel.com → 對應 project
  2. Deployments → 點選最新部署 → Functions 或 Build Logs
  3. 即時 Logs：Logs tab（僅 Backend project 有 Runtime Logs）

CLI 方式：

# 查看後端 runtime logs
cd backend
vercel logs https://edm-dashboard-api.vercel.app

6.4 更新環境變數

Dashboard 方式：

  1. 前往 Project Settings → Environment Variables
  2. 新增或修改變數
  3. 重新部署才會生效

CLI 方式：

# 新增環境變數
cd backend
vercel env add VARIABLE_NAME

# 部署使變更生效
vercel --prod

6.5 緊急恢復

⚠
Supabase 資料庫被暫停
前往 Supabase Dashboard → 點選 "Restore project" → 等待 1-2 分鐘恢復 → 驗證：curl https://edm-dashboard-api.vercel.app/api/health
✓
Vercel 部署回滾
Vercel Dashboard → Deployments → 找到上一個正常部署 → "..." → "Promote to Production"
