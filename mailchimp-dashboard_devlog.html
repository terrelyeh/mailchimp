Mailchimp Multi-Region Dashboard

從零建立一個統一的多區域電子報分析儀表板，取代原本要切四個 Mailchimp 帳號看數據的低效方式。

整理日期：2026-03-17  ·  類型：工具開發紀錄
我用 AI 做了什麼 從零建立多區域 Mailchimp 電子報分析儀表板，涵蓋前後端、認證系統、AI 分析、Docker 部署
沒有 AI 的話 以這個規模的功能，至少需要 2-3 人的小團隊花 2-3 個月開發
最終成果 完整的生產環境 Dashboard，已部署在 Zeabur，日常用於管理 4 個區域的 EDM 行銷數據
01

為什麼要做這件事

EnGenius 在全球有四個區域分公司（US / EU / APAC / JP），各自有獨立的 Mailchimp 帳號在發電子報。行銷團隊要看「全局表現」時，必須分別登入四個 Mailchimp 後台，手動比對數據。

  • 效率低：每次要看報告就要切四個帳號，光登入就浪費時間
  • 無法跨區比較：Mailchimp 原生不支援跨帳號的 Campaign 比較
  • 缺乏統一視角：沒有一個地方可以一眼看到哪個區域表現最好、哪個需要關注
  • 分享困難：想把數據分享給主管或其他部門，只能截圖貼 Slack
→
目標很明確：建一個統一的儀表板，一個畫面看全球，能比較、能分析、能分享。
02

技術選型

React 19 Vite 5.4 Tailwind CSS Recharts FastAPI SQLite Gemini AI Docker
技術選擇為什麼選它
前端框架React 19 + Vite生態成熟、元件化開發效率高、Vite 熱更新快
樣式Tailwind CSS快速迭代 UI，不用寫一堆 CSS 檔案
圖表RechartsReact 原生支援、API 直覺、客製化容易
後端框架Python FastAPI效能好、自動生成 API 文件、async 原生支援
資料庫SQLite輕量、無需額外服務、適合這個規模的資料快取
AI 分析Google Gemini 2.0 Flash支援圖片輸入（截圖分析）、速度快、成本低
認證JWT + bcrypt業界標準、無需第三方服務
部署Docker + Zeabur容器化確保一致性、Zeabur 支援 Persistent Volume
!
選型原則：優先考慮「能快速迭代」和「維護成本低」。這不是高併發系統，而是內部分析工具，SQLite 完全夠用，不需要上 PostgreSQL。
03

建了什麼功能

核心儀表板

總覽頁面 — KPI 卡片（發送量、開信率、點擊率、退訂數、訂閱人數）、AI Executive Summary、時間序列圖表、區域摘要卡片、MoM/QoQ 趨勢比較。

區域詳細頁面 — 單一區域完整 KPI、效能趨勢圖、Campaign 列表（搜尋、排序、分頁）、Audience 篩選、Segment Coverage。

跨區域 Campaign 比較

Mailchimp 原生完全做不到的功能：選擇不同區域的 Campaign 放在同一張表格裡比較，可儲存比較群組供日後檢視，搜尋功能支援跨區域找 Campaign。

AI 分析（Google Gemini）

按下浮動按鈕 → 系統自動截圖 → 送 Gemini AI 分析 → 產出結構化報告：現況診斷、核心洞察、本週執行清單（5-7 項）、自動化建議。AI 模型、Prompt、輸出格式都可在後台自訂。

認證與使用者管理

JWT 認證（24hr token）、Admin / Viewer 兩種角色、首次登入強制改密碼、管理員可新增/刪除使用者與重設密碼。

分享與匯出

分享連結：保留篩選狀態、可設密碼和過期時間、無需登入。匯出：PNG 高解析度截圖、PDF 多頁報告。

其他
  • 閾值警示（開信率/點擊率低於設定值紅色標示）
  • 排除測試 Audience，不影響統計
  • 活動日誌追蹤使用者操作
  • API 診斷工具（endpoint 狀態與回應時間）
  • 響應式設計（Desktop / Tablet / Mobile）
  • Mock 資料模式（無 API key 也能開發測試）

解決了什麼問題

原本的痛點Dashboard 怎麼解決
要切 4 個 Mailchimp 帳號看數據一個畫面看全球，KPI 自動彙總
無法跨區域比較 Campaign 表現Campaign 比較功能，跨區域放在同一張表
看完數據不知道要做什麼AI 分析直接產出診斷報告和行動清單
分享數據只能截圖一鍵產生分享連結，可設密碼保護
主管要報告要手動整理PDF/PNG 匯出，含 KPI、圖表、Campaign 列表
測試 Audience 干擾統計排除功能，一鍵排除不影響正式數據
沒有歷史趨勢可看時間序列圖表 + MoM/QoQ 自動比較
05

技術架構

使用者瀏覽器
    │
    ▼
Frontend (Nginx + React SPA)   Port 80
    │   Vite build → 靜態檔案
    │   Nginx reverse proxy → /api/*
    ▼
Backend (FastAPI + Uvicorn)   Port 8000
    │   45 API endpoints
    │   JWT 認證 middleware
    │   BackgroundTasks 背景同步
    │   ThreadPoolExecutor 平行抓取
    ├──────────────┐
    ▼               ▼
SQLite          Mailchimp API (4 regions)
/data volume     Gemini AI API (截圖分析)
→
部署在 Zeabur：Docker Compose 編排前後端兩個容器，SQLite 掛載在 Persistent Volume 確保重新部署不遺失資料。
06

如果繼續往下

  • 自動定時報告 — 每週自動產生 AI 分析報告，寄到信箱或推播到 Slack
  • A/B Test 分析 — 當 Mailchimp 有做 A/B 測試時，自動比較各變體的表現
  • 訂閱增長追蹤 — 長期追蹤各區域的訂閱人數變化趨勢
  • 與 CRM 整合 — 把 Campaign 表現和實際轉換率（Salesforce / HubSpot）串起來
07

Takeaway

核心思路

用 AI 協作的方式，一個人就能建出原本需要小團隊才能完成的內部工具。關鍵不在技術多複雜，而是清楚知道「要解決什麼問題」，然後讓 AI 處理實作細節。

可移植的部分
  • 「多帳號聚合成統一儀表板」的模式適用於任何 SaaS 工具（Google Ads、Facebook Ads、GA4 等）
  • 「截圖 → AI 分析 → 結構化報告」的流程可以套用到任何數據儀表板
  • Docker + Zeabur 的部署方式適合所有需要持久儲存的小型內部工具
!
特殊條件：Mailchimp API 相對穩定且文件齊全；SQLite 適合十幾個使用者的規模；Gemini AI 分析品質取決於 Prompt 設計，需根據實際需求調整。
